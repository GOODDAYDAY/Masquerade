# REQ-017: UI Layout Fix & AI Intelligence Upgrade

## 1. Background & Motivation

### UI 问题

1. **头像溢出**：12 名玩家头像在 SpeakingScene/ActionScene 中一行摆不开（每个 130-156px + gap，总宽超出画面中央区域），而 VotingScene（104px）却能摆开。需要自适应布局。
2. **字幕截断**：SpeakingScene 的 speech bubble 设置了 `maxHeight: 500, overflow: "hidden"`，长发言直接被裁切。Remotion 视频中无法用 CSS 滚动，需要帧驱动的自动滚动。

### AI 发言质量问题

分析 `game_werewolf_20260317_112521.json` 和 `game_werewolf_20260317_150315.json` 两局实战，发现以下系统性问题：

| 问题 | 严重度 | 典型案例 |
|:---|:---|:---|
| **回声效应** | 严重 | 一轮中"甄冲动拍桌子"被 6 人复述，每人引用相同原话 |
| **人设沦为口头禅** | 严重 | 甄推理每轮"从贝叶斯概率来看"，甄话痨每轮"哎呀终于轮到我说话了" |
| **狼人互抄话术** | 严重 | 同阵营狼人用几乎相同的论点攻击预言家，暴露协同关系 |
| **缺乏元知识** | 严重 | 狼人攻击"查验顺序太巧了"——预言家验谁是策略自由，这是无效攻击 |
| **狼人遗言自曝** | 中等 | 甄号召遗言"我是狼人甄号召"——狼人遗言不应认身份 |
| **跨轮无变化** | 中等 | 甄纠结连续 3 轮"我不确定"，无任何成长或变化 |
| **忽视新信息** | 中等 | 平安夜、新死亡等关键事件无人深入分析，一直翻旧账 |

**根因分析**：
- Prompt 要求"引用原话"但无"不要重复前人已说内容"的防护
- Optimizer 仅 2-3 句预算，人设只能靠表层口头禅体现
- 无"狼人杀元知识"共享 prompt，AI 不懂基本游戏常识
- 狼人无"话术分化"指令，天然趋同
- 无跨轮变化要求，每轮用相同模板

## 2. Goals

### UI
1. 头像行自适应：12 人场景下自动缩小头像或分多行，保证不溢出
2. 长字幕帧驱动自动滚动：文字打完后内容自动上移，确保最新文字始终可见

### AI 智能
3. 消除回声效应：每人必须提出新观点，不重复前人已说内容
4. 人设深度化：通过推理风格而非口头禅体现人设
5. 狼人话术分化：同阵营狼人必须各自不同的叙事角度
6. 注入狼人杀元知识：建立共享的游戏常识规则
7. 跨轮表达变化：禁止连续轮次使用相同开场白/句式
8. 狼人遗言策略：永远不在遗言中承认狼人身份

## 3. Non-Goals

- 不改变 Thinker → Evaluator → Optimizer 三节点架构
- 不更换 LLM 模型
- 不修改游戏规则（胜利条件、角色技能等）
- 不影响交互式 Theater 播放的现有功能

## 4. Functional Requirements

### Part A: UI Layout Fix

#### FR-A1: 头像行自适应

**Remotion 场景**（SpeakingScene / ActionScene / OpeningScene）：
- 当玩家数 ≤ 8 时，保持当前大小（inactive 130px, active 156px）
- 当玩家数 > 8 时，自动缩小头像：按比例缩放使一行可容纳所有头像
- 计算公式：`avatarSize = Math.min(defaultSize, (maxRowWidth - (playerCount - 1) * gap) / playerCount)`
- `maxRowWidth` 参考画面宽度 2560px 减去左右 padding
- active 玩家头像保持比例放大（1.2x），但基数随总人数缩放

**交互式 Theater**（对应的 components/scenes/）：
- 同理自适应，基于容器宽度

#### FR-A2: 字幕帧驱动自动滚动

**Remotion SpeakingScene**：
- 当文字内容高度超过 speech bubble 的 maxHeight 时，启用帧驱动滚动
- 滚动逻辑：文字从顶部开始逐字显示，当内容超出可视区域时，`translateY` 随帧数递增向上平移
- 滚动速度与文字打字速度同步：新文字出现时，如果已超出底部边界，自动上移使最新文字可见
- 滚动结束后停留 2 秒（120 帧）再进入下一场景
- strategy tip bubble 同理处理

**交互式 Theater**：
- speech bubble 改为 `overflow-y: auto`，浏览器自带滚动条
- 新文字出现时自动 `scrollIntoView`

### Part B: AI Intelligence Upgrade

#### FR-B1: 狼人杀元知识共享 Prompt

在 `shared_prompts.py` 中新增 `WEREWOLF_META_KNOWLEDGE`，注入到所有角色的 DAY_THINKER prompt：

```
【狼人杀元知识——所有玩家的共同常识】
1. 预言家验谁是策略自由，查验顺序不构成任何怀疑依据。攻击"查验顺序可疑"是无效论证。
2. 首夜验人是随机或基于直觉，不需要理由。
3. 好人遗言应传递关键信息；狼人遗言应继续伪装，绝不承认狼人身份。
4. "平安夜"意味着守卫守中或女巫救人，应深入分析而非一笔带过。
5. 投票是最重要的信息来源——谁投了谁，比谁说了什么更有参考价值。
6. 不要纠缠于谁"态度好"或"态度差"，关注逻辑一致性和信息链。
```

#### FR-B2: 反回声指令

在所有 DAY_THINKER prompt 中追加：

```
【反回声规则】
- 认真听取前面所有人的发言，但你的发言必须提出新的观点、新的分析角度或新的证据链
- 严禁重复前面发言者已经提到的论点或引用。如果某个证据前人已充分讨论，一句话带过即可，重点放在你的新发现
- 如果你确实没有新观点，至少从不同角度解读已有信息，或指出前人分析中的漏洞
```

在 DAY_EVALUATOR prompt 中增加评分规则：
```
- 如果发言内容与本轮前面玩家的发言高度重复（引用相同的话、提出相同的论点），扣 3 分
```

#### FR-B3: 人设深度化

修改 DAY_OPTIMIZER prompt，将"语气要符合角色人设"替换为更具体的指导：

```
【人设表达规则】
- 人设应通过推理方式和关注点体现，而非口头禅或固定开场白
- 禁止连续两轮使用相同的开场白或结尾语
- 例如：冲动型 → 快速下结论、容易被激怒改变立场；深沉型 → 先听后说、从细节切入；话痨型 → 信息量大但会扯远
- 你上几轮说过的话已提供在上下文中，确保这轮有明显不同的表达方式
```

#### FR-B4: 狼人话术分化

在 WOLF_DAY_THINKER prompt 中追加：

```
【话术分化规则】
- 你知道谁是你的狼人队友。在白天发言时，你和队友必须采用完全不同的叙事策略
- 不要引用或附和队友的论点。如果队友攻击了某人，你应该另辟蹊径或保持中立
- 具体策略分工建议：一狼主攻（引导怀疑方向）、一狼潜水（少说话装好人）、一狼搅局（制造混乱转移视线）
- 如果队友的论点被质疑，不要急于辩护——这会暴露关联
```

#### FR-B5: 狼人遗言策略

在 WOLF_DAY_OPTIMIZER 中追加遗言专项规则：

```
【狼人遗言铁律】
- 永远不要在遗言中承认自己是狼人
- 遗言应该：(a) 继续伪装好人，质疑投你的人 (b) 把水搅浑，给存活的队友制造生存空间 (c) 反咬一个好人
- 即使全场已经确认你是狼，遗言也要坚持"我是好人，你们会后悔的"
```

#### FR-B6: 跨轮变化要求

在所有 DAY_OPTIMIZER prompt 中追加：

```
【跨轮变化规则】
- 你前几轮的发言内容已提供在上下文中
- 本轮发言的开场方式、句式结构、论证角度必须与前几轮明显不同
- 如果你前轮以提问开头，这轮换成陈述；如果前轮很激动，这轮可以冷静分析
```

#### FR-B7: 新信息优先

在所有 DAY_THINKER prompt 中追加：

```
【信息新鲜度规则】
- 优先讨论本轮的新信息：谁昨晚死了？为什么？这说明什么？
- 如果昨晚是平安夜，深入分析原因（守卫守中？女巫救人？狼人空刀？）并讨论对局势的影响
- 然后才考虑回顾历史信息，且仅引用尚未被充分讨论的旧信息
```

## 5. Technical Constraints

- UI 修改影响 Remotion 和 Theater 两套组件，需同步修改
- Prompt 修改集中在 `backend/engine/werewolf/strategy.py` 和 `backend/engine/shared_prompts.py`
- Prompt 改动不能超过 token 预算（deepseek-chat 上下文窗口限制）
- 自动滚动需要在 Remotion 中用 `interpolate()` + `useCurrentFrame()` 实现，不能用 CSS scroll

## 6. Acceptance Criteria

### UI
- [ ] AC-1: 12 人场景下所有头像在一行内完整显示，无溢出或遮挡
- [ ] AC-2: 长发言（超过 5 行）在视频中自动滚动，最新文字始终可见
- [ ] AC-3: 交互式 Theater 中长发言可手动滚动

### AI 智能
- [ ] AC-4: 同一轮中不同玩家的发言不再出现大段重复引用
- [ ] AC-5: 同一玩家连续轮次的发言有明显差异（不使用相同开场白）
- [ ] AC-6: 狼人不在遗言中承认身份
- [ ] AC-7: 狼人不使用"查验顺序可疑"类无效攻击
- [ ] AC-8: 同阵营狼人的发言角度明显不同
- [ ] AC-9: 平安夜后有玩家深入分析原因
