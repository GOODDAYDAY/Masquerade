# REQ-006 Werewolf Game

> Status: Completed
> Created: 2026-03-15
> Updated: 2026-03-15

## 1. Background

Masquerade 是一个 AI 桌游对战平台，当前仅支持"谁是卧底"(Spy)。为验证架构扩展性并丰富游戏内容，需添加狼人杀游戏。

当前 `GameRunner` 中存在 Spy 游戏的硬编码耦合（直接 import spy 模块、硬编码策略选择/动作日志/投票结算逻辑），需先重构 Runner 实现真正的游戏无关，再添加狼人杀引擎。

目标：高内聚、低耦合，理想情况下仅新增 engine 子模块即可完成扩展。

## 2. Target Users & Scenarios

- **开发者：** 通过 CLI 启动狼人杀游戏（`python -m backend.main werewolf --config config/games/werewolf.yaml`），观看 AI 玩家对战
- **前端用户：** 通过 Web 回放观看狼人杀游戏录像（利用已有的通用渲染能力，前端零修改）

## 3. Functional Requirements

### F-01 重构 GameRunner 消除 Spy 耦合

- **主流程：**
  - 移除 Runner 中所有对 `backend.engine.spy` 的直接 import
  - 策略选择逻辑下沉到 Engine：将 `get_agent_strategy()` 改为接受 `player_id` 参数，由 Engine 根据玩家角色返回对应策略
  - 动作日志格式化由 Engine 提供接口（如 `format_action_log(player_id, action) -> str`）
  - 公开信息广播策略由 Engine 决定（如 `should_broadcast_action(action) -> bool`）
  - 轮次结算逻辑由 Engine 通过已有 `apply_action` + `get_public_state` 接口驱动，Runner 不再自行提取投票结果
- **错误处理：**
  - 重构后所有现有 Spy 游戏测试必须通过
  - Engine 接口变更需向后兼容或同步更新 Spy 实现
- **边界：**
  - 不修改 Agent 框架（LangGraph 层）
  - 不修改 Script 录制框架的核心结构

### F-02 狼人杀游戏引擎核心

- **角色体系：**
  - 狼人(Werewolf) — 狼人阵营，夜晚击杀村民
  - 村民(Villager) — 好人阵营，无特殊能力
  - 预言家(Seer) — 好人阵营，夜晚查验一名玩家阵营
  - 女巫(Witch) — 好人阵营，拥有一瓶解药和一瓶毒药（整局各限用一次）
  - 猎人(Hunter) — 好人阵营，死亡时可选择开枪带走一人
  - 守卫(Guard) — 好人阵营，夜晚保护一名玩家
- **配置驱动：** 通过 `config/games/werewolf.yaml` 配置角色数量和玩家信息
- **角色分配：** 随机分配角色，确保阵营配置合法（狼人数 < 好人总数）

### F-03 狼人杀游戏流程（夜晚阶段）

- **核心约束：所有夜晚交流均为动作描述（肢体动作、手势、眼神等），不能使用语言文字**
- **狼人行动：**
  - 所有狼人进行 **2 轮内部动作交流**（轮流描述动作，仅狼人之间可见）
  - 例如："指向了某个方向""比了个割喉手势""摇了摇头""竖起了大拇指"
  - 2 轮动作交流结束后，由最后行动的狼人提交击杀目标
- **守卫行动：** 描述保护动作，选择保护一名玩家。不能连续两晚保护同一人
- **预言家行动：** 描述查验动作，选择查验一名玩家，获知其阵营（好人/狼人）
- **女巫行动：**
  - 得知当晚被狼人击杀的玩家（如果有）
  - 可选使用解药（救被杀者）或毒药（毒杀一名玩家），每种整局限用一次
  - 解药和毒药不能在同一晚同时使用
- **行动顺序：** 守卫 → 狼人（含 2 轮讨论）→ 女巫 → 预言家
- **夜晚结算：** 综合所有夜晚行动结果，确定死亡玩家（不公布死因）
  - 守卫保护的玩家免受狼人袭击（守卫保护 + 女巫解药不叠加，即同时触发则目标死亡）
  - 女巫毒药直接致死，不受守卫保护

### F-04 狼人杀游戏流程（白天阶段）

- **死亡公告：** 宣布昨晚死亡玩家（可能 0 人、1 人或 2 人）
- **遗言环节：** 被淘汰玩家可发表遗言
  - 猎人死亡时可选择开枪带走一名存活玩家
- **自由讨论：** 存活玩家依次发言（正常语言文字）
- **投票放逐：** 所有存活玩家投票，得票最多者被放逐
- **平票处理：** 平票则无人被放逐，直接进入下一个夜晚

### F-05 胜负判定

- **好人阵营胜利条件：** 所有狼人被淘汰（预言家、女巫、猎人、守卫、村民均为好人阵营）
- **狼人阵营胜利条件：** 存活的狼人数量 ≥ 存活的好人数量
- **检查时机：** 夜晚结算后 + 白天投票放逐后 + 猎人开枪后，均需检查胜负

### F-06 LLM Agent 策略

- **核心约束：夜晚阶段所有角色的策略 prompt 必须引导 LLM 输出动作描述，而非语言对话**
- **狼人策略：**
  - 夜晚：用肢体动作/手势暗示击杀目标和伪装策略
  - 白天：如何伪装好人、如何引导投票方向
- **预言家策略：**
  - 夜晚：查验动作描述
  - 白天：如何引导讨论、何时公布查验结果、如何自证身份
- **女巫策略：**
  - 夜晚：用药决策的动作描述
  - 白天：是否暴露身份、如何利用药物信息
- **村民策略：** 白天如何分析发言逻辑、识别狼人
- **守卫策略：** 夜晚保护目标的选择动作、白天如何参与讨论
- **猎人策略：** 死亡时是否开枪、目标选择的判断

### F-07 游戏配置文件

- 创建 `config/games/werewolf.yaml`
- 配置项：
  - 角色数量（werewolf_count, villager_count, 各特殊角色 0/1）
  - 玩家列表（name, persona, appearance）
- 验证规则：狼人数 < 好人总数，玩家总数 = 角色总数

## 4. Non-functional Requirements

- **NF-01 兼容性：** 不修改 Agent 框架（LangGraph 层）、不修改 Script 录制框架核心结构、前端零修改
- **NF-02 可测试性：** 游戏引擎可脱离 LLM 独立测试（纯状态机逻辑）
- **NF-03 代码质量：** 充分的日志记录，关键逻辑有注释

## 5. Out of Scope

- 警长竞选机制
- 丘比特、狐狸等扩展角色
- 狼人自爆机制
- 前端狼人杀专用 UI 组件
- 守卫和女巫解药同时作用于同一目标的"同守同救"高级规则变体（本版本采用简单规则：同时触发则目标死亡）
